Module Scripts

Pass values, functions and tables in one script and access it everywhere! All module scripts inside of ReplicatedStorage will be able to be accessed in all local scripts. All module scripts in ServerScriptService can be accessed in all server scripts.

What are Module Scripts?

Module scripts are very useful scripts used by many developers on Roblox. The primary use of those scripts is to run a function. Before you ask, yes it is possible to run a function in a normal script. The only problem with this is that the more and more you use it, it gets very messy. Module scripts help keep that function, along with values (settings, levels, etc), and tables with information!

How can I use Module Scripts?

Using module scripts is easy once you get used to using them! An easy way to get started is by testing code! Let's begin!

1. On the right side of "ServerScriptService", click the + button

2. In the objects menu, select "ModuleScript".

3. Remove all the current code inside of the script.

4. We must open the module script! The code below defines a variable with an empty table and returns the table in the last line. All the other code will run in between these 2 lines.

  1.     local testmodule = {}
  2.     return testmodule
Copy to Clipboard

5. Now that we actually have some code, let's create a function! The function below is a simple function which prints Hello Robloxians in the output!

  1.     local testmodule ={}
  3.     function printHello.testModule()
  4.         print("Hello Robloxians!")
  5.     end
  7.     return testModule
    8.
Copy to Clipboard

6. We are done with the module script for now. What this script is doing is it's holding a function which can be run in any server script. It helps keep the code organized in one script! You can hold as many functions and tables as you'd like in just one module script.

7. Create a new Server Script inside of Server Script Serivce.

8. Clear all the current code!

9. The first thing we must do is require the module script using a variable like the one below!

  1.     local moduleScript = require(game:GetService("ServerScriptService"):WaitForChild("ModuleScript") -- You must require the module script in ServerScriptService. Change if you renamed your script!

10. Now we can print the function!

  1.     local moduleScript = require(game:GetService("ServerScriptService"):WaitForChild("ModuleScript")
  2.     moduleScript.testModule()
    3.
